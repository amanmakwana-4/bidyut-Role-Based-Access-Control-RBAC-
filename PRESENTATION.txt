================================================================================
HOW AUTHENTICATION WORKS
================================================================================

1. USER REGISTERS
   ├─ User fills: Name, Email, Password, Confirm Password
   ├─ Frontend validates: Passwords match, min 6 chars
   ├─ Backend validates: Email unique, requirements met
   ├─ Password gets hashed with bcrypt
   ├─ User saved to MongoDB
   └─ User redirected to login page (no auto-login)

2. USER LOGIN
   ├─ User enters: Email and Password
   ├─ Backend verifies credentials
   ├─ Generates JWT token with 24-hour expiry
   ├─ Stores token in:
   │  ├─ localStorage (for API requests)
   │  └─ httpOnly secure cookie (safe from XSS)
   ├─ Returns user data with role
   └─ Frontend redirects based on role:
      ├─ Admin → /all-tasks
      └─ User → /dashboard

3. PROTECTED REQUESTS
   ├─ Every API call includes Authorization header: "Bearer token"
   ├─ Backend middleware verifies token signature
   ├─ Extracts user ID from token
   ├─ Fetches user from database
   ├─ Checks role for authorization
   └─ Allows/blocks request accordingly

4. TOKEN EXPIRATION
   ├─ Token expires after 24 hours
   ├─ Axios interceptor detects 401 response
   ├─ Clears localStorage and cookies
   ├─ Redirects user to /login
   └─ User must login again

SECURITY FEATURES
================================================================================

✓ JWT TOKENS
  - 24-hour expiration
  - Signed with secret key
  - Verified on every request

✓ PASSWORD SECURITY
  - Hashed with bcrypt (10 rounds)
  - Passwords never stored in plain text
  - Salt generated automatically

✓ SECURE COOKIES
  - httpOnly: Cannot be accessed via JavaScript
  - Secure: Only sent over HTTPS
  - sameSite: Prevents CSRF attacks

✓ PROTECTED ROUTES
  - Frontend checks authentication
  - Auto-redirects unauthenticated users
  - Admin routes check role

✓ API PROTECTION
  - Token verification middleware
  - Role-based authorization

  - Error handling

✓ DATABASE SECURITY
  - MongoDB Atlas with encryption
  - Connection string in environment variables
